# Migration plan: `.docs/` → Sphinx Stack 2.0

This plan migrates the charmlibs documentation in [`.docs/`](.docs) from the old
`sphinx-docs-starter-pack` (the `.sphinx/` era) to **Sphinx Stack 2.0**, using the
reference checkout under [`.local/sphinx-stack/`](.local/sphinx-stack) as the target.

The guiding principle: **adopt Stack 2.0's configuration surface and toolchain, but
preserve all charmlibs-specific machinery** — the per-package multi-pass build, the local
Sphinx extensions, and the diataxis preprocessor. Stack 2.0 is a *starter pack*, not a
hard framework; charmlibs deliberately diverges from its `Makefile`/`requirements.txt`
build flow in favour of a `just` + `uv` orchestration. That divergence stays.

## Constraints and non-goals

- **Keep the `.docs/` directory name.** Stack 2.0 ships docs under `docs/`; charmlibs uses
  `.docs/` (referenced by [`docs.just`](docs.just), [`.readthedocs.yaml`](.readthedocs.yaml),
  [`.github/workflows/docs.yaml`](.github/workflows/docs.yaml), and the spellcheck action).
  Migrating the *contents*, not renaming the directory.
- **Keep the `just`/`uv` multi-pass build.** Do not adopt Stack 2.0's `Makefile`/`.venv`/
  `pip` flow. The per-package autodoc build in [`docs.just`](docs.just) is essential and
  has no equivalent in the starter pack.
- **Preserve local extensions and their tests** ([`.docs/extensions/`](.docs/extensions),
  [`.docs/scripts/`](.docs/scripts), [`.docs/tests/`](.docs/tests)).
- **No content rewrites** beyond what the toolchain change forces (e.g. Vale findings).
- Output stays `dirhtml`; published URL and RTD project unchanged.

## Key breaking changes to absorb

1. **`.sphinx/` → `_dev/` rename.** Stack 2.0 renamed the dev directory. charmlibs keeps
   project files under `.sphinx/` (`requirements.txt`, `_static/`, `spellingcheck.yaml`,
   `.wordlist.txt`, `get_vale_conf.py`, `metrics/`, `pa11y.json`). Decide: rename to
   `_dev/` to match upstream (recommended, eases future `update_sp.py` syncs) or keep
   `.sphinx/`. If renamed, update `conf.py` (`html_static_path`, `templates_path`),
   [`docs.just`](docs.just), [`.gitignore`](.docs/.gitignore), `spellingcheck.yaml`
   paths, and the docs workflow.
2. **`canonical-sphinx` unbundling (`[full]` → `~=0.6`).** The biggest risk. Today
   `canonical_sphinx` auto-enables: `custom-rst-roles`, `myst_parser`, `notfound.extension`,
   `related-links`, `sphinx_copybutton`, `sphinx_design`, `sphinx_reredirects`,
   `sphinx_tabs.tabs`, `sphinxcontrib.jquery`, `sphinxext.opengraph`, `terminal-output`,
   `youtube-links`. Under 0.6 these must be declared explicitly in both
   `requirements.txt` and the `extensions` list (see Stack 2.0
   [`conf.py`](.local/sphinx-stack/docs/conf.py)). Map each old auto-extension to its new
   standalone package.
3. **Style/spell/lint toolchain swap.** Old: `pyspelling` + aspell
   ([`spellingcheck.yaml`](.docs/.sphinx/spellingcheck.yaml)). New: **Vale** for spelling/
   inclusive-language/style, `pymarkdownlnt` for Markdown, `pa11y` for accessibility. The
   charmlibs spellcheck currently runs via `just docs spelling` (pyspelling) and the
   `canonical/documentation-workflows/spellcheck` action. Migrating to Vale changes the
   wordlist mechanism (`.custom_wordlist.txt` is appended to the Canonical Vale vocabulary
   `accept.txt`, replacing `.sphinx/.wordlist.txt`).
4. **Redirects engine.** Stack 2.0 defaults to `sphinx-rerediraffe`
   (`rediraffe_redirects = "redirects.txt"`). charmlibs uses `sphinx_reredirects` with an
   empty `redirects = {}`. Low risk (empty today) — can keep `sphinx_reredirects` or adopt
   the new default; pick one and remove the other.
5. **`sphinx-llm` / `llms.txt`.** New optional feature. Decide whether to enable it
   (adds `llms_txt_description`, `markdown_http_base`). Recommended on.

## Risks & mitigations

- **Hidden reliance on bundled extensions.** Mitigate by grepping content for directive/
  role usage and enabling each explicitly; build with `-W` to surface unknowns.
- **Vale noise on a large existing corpus.** Mitigate by seeding `.custom_wordlist.txt`
  from the existing wordlists and staging Vale as non-blocking first if needed.
- **canonical-sphinx 0.4→0.6 theme/template changes.** Mitigate with visual HTML diffing;
  pull Stack 2.0 `_templates/` only if 0.6 requires them.
- **Path churn from `.sphinx/`→`_dev/`.** Mitigate by doing the rename in one focused
  commit with a global path update + green build.
